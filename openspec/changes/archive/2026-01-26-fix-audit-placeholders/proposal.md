# Proposal: Fix Audit Placeholders

## Summary

处理Danci全功能审计中发现的占位符功能。本提案聚焦于P3级别的18个占位符功能，这些功能已有UI但后端返回mock数据或TODO响应。

## Motivation

### Problem Statement

2025-01-25全功能审计发现18个P3占位符功能：

**管理后台 (10)**

1. **LLM顾问分析** - 返回hardcoded建议
2. **AMAS可解释性** - 所有参数返回固定值
3. **因果推理ATE计算** - 返回"disabled"
4. **聚类可视化** - 返回空聚类列表
5. **内容增强预览** - 返回NOT_IMPLEMENTED
6. **问题自动修复** - 返回NOT_IMPLEMENTED
7. **变体审批流程** - 返回NOT_IMPLEMENTED
8. **优化效果预测** - 返回固定预测值
9. **系统设置保存** - 返回成功但无实际存储
10. **告警规则管理** - 返回hardcoded规则

**用户功能 (5)** 11. **学习建议生成** - 返回固定模板建议 12. **个性化推荐** - 返回随机推荐 13. **成就系统** - 返回hardcoded成就14. **学习报告生成** - 返回模板报告 15. **社交功能** - 返回"coming soon"

**数据分析 (3)** 16. **用户分群分析** - 返回固定分群 17. **留存率计算** - 返回模拟数据 18. **学习效果预测** - 返回占位预测

### Impact

- **管理员信任**: mock数据可能导致错误决策
- **用户预期**: 占位功能创造虚假功能预期
- **代码质量**: 占位代码增加维护负担

## Proposed Solution

### 策略A: 实现真实功能

对于有明确需求和实现路径的功能，完成真实实现。

### 策略B: 明确标记为"计划中"

对于复杂度高或需求不明确的功能，UI明确标记为"计划中"而非返回假数据。

### 策略C: 移除功能入口

对于低优先级或长期不会实现的功能，移除UI入口。

### 功能分类

**实现真实功能 (策略A) - 8个**

- 用户分群分析
- 留存率计算
- 学习建议生成
- 学习报告生成
- 系统设置保存
- 聚类可视化
- AMAS可解释性（部分）
- LLM顾问分析（基础版）

**明确标记计划中 (策略B) - 7个**

- 因果推理ATE计算
- 内容增强预览
- 问题自动修复
- 变体审批流程
- 优化效果预测
- 学习效果预测
- 告警规则管理

**移除功能入口 (策略C) - 3个**

- 成就系统（重新设计后再上线）
- 社交功能（需求待定）
- 个性化推荐（与现有推荐整合）

## Scope

### In Scope

- 18个P3占位符功能的处理
- 相应UI调整
- 必要的后端实现

### Out of Scope

- P0/P1/P2级别问题
- 完全新功能开发
- 大规模重构

## Dependencies

- fix-audit-critical-issues提案（后端服务基础）
- fix-audit-missing-features提案（相关服务）

## Risks

| Risk               | Mitigation                     |
| ------------------ | ------------------------------ |
| 用户对功能移除不满 | 明确说明"计划中"，提供反馈渠道 |
| 实现复杂度评估不准 | 先实现基础版本，迭代改进       |

## Success Criteria

1. 无任何后端返回hardcoded mock数据
2. 所有"计划中"功能有明确的UI标识
3. 移除的功能入口无残留
4. 真实功能返回正确数据
