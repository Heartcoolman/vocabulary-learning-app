# G3 动画曲线设计文档

## 1. 设计理念：自然与流体
G3 动画曲线的核心设计理念是**模拟真实世界的物理运动**。在现实世界中，物体不会瞬间开始移动，也不会瞬间停止，甚至加速度也不会瞬间改变（除了碰撞）。G3 连续性消除了“加加速度”（Jerk）的突变，使得运动极其柔和、顺滑。

### 关键词
*   **Fluid (流体)**: 像水流一样自然流动，无生硬转折。
*   **Responsive (响应式)**: 动画应感觉像是用户手指的延伸。
*   **Premium (高级感)**: 细腻的过渡暗示着高品质的软件工艺。

## 2. 适用场景

### 2.1 适合使用 G3 曲线的场景
*   **全屏过渡**: 页面切换、模态框（Modal）弹出/关闭。
*   **大型组件运动**:侧边栏（Drawer）、底板（Bottom Sheet）的展开。
*   **跟手交互**: 拖拽、滑动列表（Scroll）、手势操作。
*   **强调性微交互**: 重要的点赞动画、支付成功确认。

### 2.2 不适合使用 G3 曲线的场景
*   **高频反馈**: 键盘打字、按钮点击的瞬间反馈（需要极快响应，G3 的起步可能略显“拖沓”）。
*   **极短动画**: < 200ms 的动画，人眼难以分辨 G2 与 G3 的区别，使用标准 Ease-out 即可。

## 3. 曲线特征与参数调优

### 3.1 视觉特征
*   **起步 (Ease-in)**: 极其缓慢地启动，几乎感觉不到开始的瞬间。
*   **中间 (Duration)**: 平滑加速到最大速度，然后平滑减速。
*   **结束 (Ease-out)**: 像羽毛落地一样轻柔地停止，没有明显的“刹车感”。

### 3.2 参数建议 (以 Cubic Bezier 近似为例)
虽然 Bezier 达不到完美 G3，但以下参数可作为设计参考：

*   **标准 G3 近似 (通用)**: `cubic-bezier(0.2, 0, 0, 1)` —— 强调极快的响应和极慢的结束。
*   **HyperOS 风格 (自然)**: `cubic-bezier(0.4, 0, 0.2, 1)` —— 更加平衡，类似物理弹簧。
*   **强回弹 (Bouncy)**: 使用弹簧物理参数 `Stiffness: 200, Damping: 15`。

## 4. 设计规范集成
建议将 G3 曲线定义为设计系统中的标准 Token：

| Token Name | Value (CSS/Bezier) | Usage |
| :--- | :--- | :--- |
| `ease-g3-standard` | `linear(...)` / `cubic-bezier(0.2, 0, 0, 1)` | 通用页面过渡，卡片展开 |
| `ease-g3-enter` | `cubic-bezier(0.05, 0.7, 0.1, 1)` | 元素进场，强调减速过程 |
| `ease-g3-exit` | `cubic-bezier(0.3, 0, 0.8, 0.15)` | 元素离场，强调加速过程 |

## 5. 总结
G3 动画不仅仅是数学上的平滑，更是一种**情感化设计**。它传达了软件的精致与对用户体验的尊重。在设计中应优先考虑“感觉”而非死板的参数，通过不断的真机测试（On-device Testing）来微调曲线手感。
