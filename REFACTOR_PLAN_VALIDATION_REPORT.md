### 5. React Query可能不适合当前项目

**架构评估**: 62/100 (部分过度设计)

**关键发现**:
1. ✅ **项目规模较小** (35K行前端代码)
2. ✅ **API调用模式简单** (70% CRUD，30%流程)
3. ✅ **已有良好缓存** (StorageService 5分钟TTL)
4. ✅ **学习流程是"流程密集型"**，非"数据密集型"
5. ❌ **不具备React Query典型场景**: 无大量并发请求、复杂数据依赖、实时同步

**问题**:
- useMasteryLearning是**有状态流程**，不适合查询模式
- AMAS状态需要精确控制时序，React Query反而增加复杂度
- 引入26KB依赖，但边际收益小

**建议**:
- 🎯 **优化现有StorageService** 而非全面引入React Query
- 🎯 仅在简单API使用React Query，复杂流程保持现状
- 🎯 Zustand仅用于Toast和UI状态，不拆分6个Store
